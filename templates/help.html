How this works

This tool helps you author a course outline with reference to the official curriculum hierarchies (an AP CED, the IB syllabus, a book). You draft your own raw objectives, map them onto those references so you can see what's covered and what's missing, and organize them into the course outline (units and lessons).

Load a reference→ Draft objectives→ Map coverage→ Build the outline→ Download the plan

The pieces

Course
The top-level thing we’re planning (CSA, CSP, IB). Each course has a reference hierarchy and, once we start planning, an outline.
Reference hierarchy
The official outline — an AP Course & Exam Description or the IB syllabus — loaded from a markdown file and treated as read-only. Its leaves (the deepest items, e.g. essential-knowledge statements) are the things you must cover.
Raw objectives
Granular statements of things students should be able to do. Raw objectives can be associated with one or more hierarchies and provide the tracing between the reference hierarchy for a course and its course outline.
Coverage
A link saying “this objective covers that reference leaf.” One objective can cover several leaves and a leaf can be covered by several objectives.
Course outline
The outline of the course, made up of units that contain lessons. Raw objectives should eventually be associated with specific lessons so we can link the lesson to the parts of the reference hierarchy that that raw objective is linked to.
Learning objective
The objective for a single lesson, the thing we’d write on the whiteboard at the beginning of class.

Reading coverage

Every hierarchy page (a reference or the course outline) has the same shape: a tree of nodes on the left with a drop zone under each, the raw-objective pool on the right, and a coverage bar across the top counting its leaf nodes, how many are covered (have at least one objective on them) and how many are still uncovered — the gaps. A node with objectives beneath it is covered; an empty drop zone is a gap.

An objective moves through a lifecycle: drafted in the pool → mapped onto the reference leaves it covers → placed in a lesson of the course outline. The downloadable plan (below) reports, per reference leaf, the lesson(s) that reach it and the leaves still uncovered.

The pages

Under each course in the sidebar:

Course outline
The outline you're authoring (the first item under each course): create units, add lessons, and drag raw objectives from the pool into a lesson — or onto a unit's rough zone if you haven't picked the lesson yet. Give each lesson its learning objective. Plan (.md) downloads the rendered deliverable.
Objectives
A compact, sortable table of every raw objective in the course, with a column per hierarchy showing the node ids it covers — your at-a-glance view of what's mapped where (the id pills are read-only; hover for the node's text). Edit objective text inline; add new ones; upload/download the set.
Reference hierarchies
Each official hierarchy (CED, book, syllabus) as a node tree with the raw-objective pool beside it. Drag an objective from the pool onto the node it covers (or write a new one in the pool and drag it in); drag between nodes to recategorize. The tree itself is fixed — it comes from the source.

And in the sidebar footer, two course-independent pages: Help (this page) and Settings (restore from version control). Create a course with + next to the title; each course in the sidebar has its own controls (see below).

Adding material in the app

  1. Draft an objective. Type one into the New objective… box at the top of the pool on any hierarchy page, or use the + objective link under a node to add one already mapped/placed there; or add one on the Objectives page.
  2. Map coverage. On a reference hierarchy page, drag an objective from the pool onto the node it covers; drag it between nodes to recategorize. The Objectives page then shows that mapping in the hierarchy's column.
  3. Build the outline. On the Course outline page, “Add unit”, then + lesson, then drag raws from the pool into lessons (or a unit's rough zone). Write each lesson's learning objective in its italic field.
  4. Everything auto-saves to the live working database as you type or drag — there are no Save buttons.
Edits go to the live database immediately. To capture a reviewable, committable state, click the save icon on the course in the sidebar (greyed when the corpus is already up to date) — it writes the course's git-diffable plan.md (the outline) plus objectives.tsv / coverage.tsv into its corpus directory.

Setting up a new course

Drive setup from the sidebar:

  1. Create the course. Click + next to Lesson Planning, enter a course id and title.
  2. Add a reference hierarchy. Click the + in the course's sidebar header and upload a hierarchy markdown file (hand-authored, or produced by the hierarchy-extractors repo from a PDF/book). The flavor is detected from its headings; title/kind come from its front matter; the slug defaults to <course>-<kind>. Add as many as you need.
  3. Seed objectives (optional). On the Objectives page, upload a plain-text list (one objective per line) or a TSV with an objective column to bulk-add raw objectives to the pool. (Any node-id column is ignored here — categorize on a hierarchy page by dragging an objective onto the node.)
  4. Then plan in the app as above.

The sidebar has the rest of a course's controls: click its title to rename it; on each reference, a ★ sets the primary (shown only with more than one) and a trash deletes it; the course header has a download (export the whole course as a bundle file) and a trash (delete the course). Recreate a course elsewhere with …or import a course bundle under the top +.

Equivalent from a terminal: uv run load_nodes.py my-course-hierarchy.md db.db and uv run import_objectives.py <objectives.tsv> db.db --course mycourse.

Saving & version control

The database (db.db) is the live working copy and is not checked in. The committed state is the corpus: a directory of course directories, each holding its hierarchy markdown, plan.md, and objectives.tsv / coverage.tsv. The course's save icon writes it back to the corpus. Starting the app with no database creates an empty one and loads any course already in the corpus; the Settings page's Restore from version control reloads every course from disk — so the whole thing is reproducible from version-controlled markdown. (uv run rebuild_db.py does the same from a terminal, rebuilding from scratch.)

Get the finished plan out with Plan (.md) on a course's Course outline page — the rendered deliverable (units, lessons, traceability, and gaps) as markdown.
